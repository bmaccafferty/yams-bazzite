#+title: YAMS-Bazzite: Yet Another Media Server (for Fedora Atomic / Bazzite)

A fork of [[https://github.com/rogsme/yams][YAMS]] adapted for [[https://bazzite.gg][Bazzite]] and other Fedora Atomic-based distros. The underlying Docker Compose stack is identical — only the installer and CLI path were changed to work on immutable systems.

* What Changed

| Original (YAMS) | YAMS-Bazzite |
|---|---|
| Install deps via ~apt~ | Install deps via ~dnf~ |
| Ubuntu snap Docker detection | Removed (not relevant) |
| Automatic Docker install (Debian script) | Removed — Docker must be installed manually via ~rpm-ostree install docker~ |
| CLI installed to ~/usr/local/bin/yams~ (requires ~sudo~) | CLI installed to ~~/.local/bin/yams~ (no sudo, already in PATH) |
| ~docker.sh~ (Debian-specific Docker installer) | No-op stub |

** Files Modified

- [[./install.sh]] — 5 changes: ~apt~ → ~dnf~, removed snap check, removed Debian Docker installer path, CLI path → ~~/.local/bin~
- [[./docs.org]] — Same changes mirrored (literate programming source for ~install.sh~)
- [[./docker.sh]] — Replaced with a no-op stub

** Why These Changes

Bazzite is an immutable Fedora Atomic distro. Unlike Debian/Ubuntu:
- ~/usr/local/bin~ is part of the read-only root — can't install there without overlay
- Package manager is ~dnf~ / ~rpm-ostree~, not ~apt~
- Docker must be layered via ~rpm-ostree~ (or use the built-in ~podman~ with compatibility layer)
- No snap package manager

* Prerequisites

1. Docker installed:
   #+begin_src bash
   rpm-ostree install docker docker-compose-plugin
   systemctl reboot
   #+end_src

2. Your user in the ~docker~ group:
   #+begin_src bash
   sudo usermod -aG docker $USER
   # then log out and back in
   #+end_src

3. Firewall ports open for the web UIs:
   #+begin_src bash
   sudo firewall-cmd --add-port={8080,8081,8096,8989,7878,8686,6767,9696,9000}/tcp --permanent
   sudo firewall-cmd --reload
   #+end_src

4. A media directory (default: ~/srv/media~)

5. A VPN provider (optional) — supported providers listed at [[https://yams.media/advanced/vpn]]

* Installation

#+begin_src bash
git clone --depth=1 https://github.com/bmaccafferty/yams-bazzite.git /tmp/yams-bazzite
cd /tmp/yams-bazzite
bash install.sh
#+end_src

Follow the interactive prompts. The installer will:
1. Ask for your user, install paths, media service (Jellyfin/Emby/Plex)
2. Configure VPN (or skip)
3. Copy and configure the Docker Compose file
4. Start all containers
5. Install the ~yams~ CLI to ~~/.local/bin/yams~

* Post-Installation

Your services will be available at:

| Service | URL |
|---|---|
| Jellyfin/Emby | ~http://HOST_IP:8096~ |
| Plex | ~http://HOST_IP:32400/web~ |
| qBittorrent | ~http://HOST_IP:8081~ |
| SABnzbd | ~http://HOST_IP:8080~ |
| Sonarr | ~http://HOST_IP:8989~ |
| Radarr | ~http://HOST_IP:7878~ |
| Lidarr | ~http://HOST_IP:8686~ |
| Bazarr | ~http://HOST_IP:6767~ |
| Prowlarr | ~http://HOST_IP:9696~ |
| Portainer | ~http://HOST_IP:9000~ |

Manage the stack with ~yams~:
#+begin_src
yams start            # Start all services
yams status           # Check service status
yams stop jellyfin    # Stop a specific service
yams restart          # Restart all services
yams check-vpn        # Verify VPN is masking qBittorrent IP
yams backup /path     # Backup to tar.gz
yams destroy          # Remove all containers and volumes
#+end_src

* SELinux

If containers can't read mounted volumes, add ~:z~ to the volume mounts in ~docker-compose.yaml~:
#+begin_src
volumes:
  - /srv/media:/data:z
#+end_src
